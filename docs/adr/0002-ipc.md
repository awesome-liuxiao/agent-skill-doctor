# ADR 0002: Local IPC

Status: accepted for Milestone 1.

The on-demand worker uses authenticated local `multiprocessing.connection`
transports: Unix-domain sockets on POSIX and named pipes on Windows. Transport
authentication uses a domain-separated key derived from the OS-protected state
master key. The socket directory is mode `0700` and the socket is mode `0600`;
named-pipe access additionally requires possession of the per-user auth key.

Only length-bounded UTF-8 JSON is sent with `send_bytes` and `recv_bytes`; the
pickle-based `send` and `recv` APIs are prohibited. Every request and response
has a protocol version and request identifier. The worker serializes job
execution, persists events and state transitions before acknowledging them,
recovers interrupted jobs as resumable failures, and exits after five idle
minutes by default.
